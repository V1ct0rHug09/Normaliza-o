Introdução

A Normalização é focada na prevenção de problemas com repetição e atualização de dados, assim como o cuidado com a integridade dos dados. Consiste em um processo de análise de uma relação para assegurar que o banco de dados seja bem formado, que tenham atribuítos coerêntes e corretos para o tipo de dados que vai armazenar. É o decompor das relações com as anomalias de atualização para produzir relações menores e bem estruturadas. Mediante a decomposição das relações presentes no banco de dados, este processo busca minimizar anomalias, isto é, repetições e redundâncias entre os dados. Ao identificar anomalias, o conjunto de regras da normalização tentará eliminá-las e redefinir as relações afetadas, para que tudo se encaixe em seu devido lugar depois das alterações. Ou seja em uma relação de normalização de banco de dados podemos inserir, excluir ou modificar dados sem gerar anomalias, seu objetivo é minimizar redundâncias e anormalias de inclusão, exclusão e modificação.

Desenvolvimento

Este conceito foi apresentado originalmente em um artigo científico publicado pela IBM de autoria do matemático Edgar F. Codd, intitulado "Um modelo de dados relacionais para grandes bancos de dados compartilhados" (1970). Codd se centra nos valores dos elementos relacionados no banco de dados, não em ligações ou agrupamentos específicos.


As formas normais, existem várias dessas regras para garantir o sucesso da estruturação do nosso banco de dados.
Porém três delas são as mais utilizadas na normalização:

1ºForma de Normalização (1FN)
A 1FN exige que a tabela esteja em formato tabular, onde todos os atributos são atômicos (não divisíveis). Isso significa que cada coluna deve conter um único valor, e cada linha deve ser única.
2ºForma de Normalização (2NF) A 2NF requer que a tabela esteja na 1NF e que todos os atributos não-chave sejam totalmente dependentes da chave primária. Isso elimina dependências parciais.
3º Forma de Normalização A 3NF requer que a tabela esteja na 2NF e que não haja dependências transitivas (um atributo não-chave depender de outro atributo não-chave).

1º Primeira Forma de Normalização (1FN)

Em 1NF, cada célula de uma tabela deve conter um único valor atômico, ou seja, não pode conter valores compostos ou repetidos. Por exemplo, vamos supor que temos uma tabela de cliente com colunas para código do cliente, nome, telefone e endereço do cliente. 3º Terceira Forma de Normalização (3FN)

Na terceira forma normal trabalhamos precisamente com a organização dos atributos que dependem uns dos outros, porém que não são atributos chaves (primárias ou estrangeiras). Caso necessário, é criada uma tabela secundária para reestruturar a relação de dependência entre os atributos. Essas tabelas devem ter a chave primária ou estrangeira. Conclusão

A normalização de bancos de dados é essencial para organizar dados de forma eficiente, reduzindo redundâncias e evitando inconsistências. Ao aplicar as formas normais, melhora-se a integridade, segurança nas operações e facilita-se a manutenção e escalabilidade do sistema. No entanto, o uso excessivo da normalização pode aumentar a complexidade das consultas e o número de tabelas, devendo ser equilibrado com as necessidades de desempenho do sistema.
Aprender a Normalização é um passo importante para quem está começando a modelar um banco de dados e certamente resulta em uma maior eficiência na hora de abstrair o banco e seus atributos.
